ft_putchar.c

Write a function that prints a single character to stdout

System calls File descriptors Address-of operator
üß† Logic & Approach:

Understanding write() parameters:

  • First parameter 1 = file descriptor for stdout
  • Second parameter &c = memory address of the character

üí° Key Insight: Think of write() like mailing a letter - you need the destination (fd=1 screen), the content's location (&c), and how many pages (1 byte).

ex01 ‚Äî ft_print_alphabet

ft_print_alphabet.c

Print the alphabet from 'a' to 'z'

While loops Character incrementation ASCII values
üß† Logic & Approach:

Characters are numbers! In ASCII, 'a' = 97, 'b' = 98, ..., 'z' = 122

The algorithm:

  • Start with char i = 'a'
  • Loop while i <= 'z'
  • Print and increment: i++
üí° Key Insight: char is just a small integer. When you do i++ on a char, you're adding 1 to its ASCII value!
EASY

ex02 ‚Äî ft_print_reverse_alphabet

ft_print_reverse_alphabet.c

Print the alphabet in reverse (z to a)

üß† Logic & Approach:

Same concept, opposite direction:

  • Start with char c = 'z'
  • Loop condition: while (c >= 'a')
  • Decrement: c--
üí° Key Insight: If ++ moves forward in ASCII, then -- moves backward. 'z' - 1 = 'y'!

üß† Three Phases of Parsing:

ex08 ‚Äî ft_ten_queens_puzzle ‚≠ê‚≠ê‚≠ê

ft_ten_queens_puzzle.c

Place 10 queens on 10√ó10 board with no conflicts, count all solutions

Backtracking Constraint checking Recursion
üß† Solution Breakdown:
1. Board Representation

int board[10]; // board[col] = row where queen is

This automatically prevents column conflicts!

2. Validity Check
  • Same row: board[i] == row
  • Same diagonal: |board[i] - row| == |i - col|
3. Backtracking Algorithm
for each row 0-9:
    if is_valid(row, col):
        board[col] = row      // Place queen
        solve(col + 1)        // Try next column
        board[col] = -1       // Backtrack
üí° Answer: There are 724 solutions for the 10-queens puzzle!
